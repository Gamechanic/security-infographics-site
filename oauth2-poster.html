01
The problem OAuth solves — stop giving apps your password

OAuth lets you delegate limited access — the app acts for you without ever seeing your password.

02
The four roles — who's who in every OAuth flow

👤 Resource Owner

You — the user who owns the data and grants access.

📱 Client

The app that wants access — web, mobile, SPA, or service.

🔐 Authorization Server

Verifies you & issues tokens. /authorize + /token.

🗄️ Resource Server

The API holding your data. Accepts the access token.

Client asks · Authorization Server approves · Resource Server serves.

03
The Authorization Code flow — the modern default, with PKCE

The app only ever holds a code, then a token — your password never leaves the Authorization Server.

04
Authorization ≠ Authentication
🎟️
OAuth 2.0 = authorization — “what may this app do?” It grants scoped access to an API.
🪪
OIDC = authentication — “who is this user?” A thin layer on OAuth that adds an ID token.

Using an access token to “log in” is the classic mistake — that's OIDC's job.

05
The tokens

Access

Short-lived. Sent as Bearer to call APIs.expires in minutes

Refresh

Long-lived. Silently mints new access tokens.store securely

ID (OIDC)

Proves who logged in. Not for calling APIs.a JWT

Bearer = whoever holds it, uses it. Guard access tokens.

06
Scopes & consent

Scopes are the permissions an app requests — the list on the consent screen you approve.

read:contactsrepoemailoffline_access

Least privilege — ask for the narrowest scopes you need.

08
On the wire — exchange the code, then call the API

// ← response
{ "access_token": "eyJhbGciOi…", "token_type": "Bearer", "expires_in": 3600 }
// 2 · call the resource server with the token
GET /v1/contacts   Authorization: Bearer eyJhbGciOi…
09
PKCE — say “pixie”

Stops a stolen authorization code from being redeemed by an attacker.

hash a secret → challenge→ send on /authorize
reveal verifier on /token→ server checks the match

OAuth 2.1 makes PKCE mandatory for every client.

10
Watch out
🎭
state param — ties the response to your request. Skipping it invites CSRF.
↩️
Exact redirect_uri — loose matching = open redirect & stolen codes.
🥷
Token theft — bearer tokens are cash. Short expiry, HTTPS-only, never log them.
🚫
No implicit flow — tokens in the URL leak. Use Auth Code + PKCE.
11
The takeaway
OAuth 2.0 is delegated access — scoped, expiring, revocable — so an app can act for you without ever holding your password.
Redirect→ Consent→ Code→ Token→ Call API
